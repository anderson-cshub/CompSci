Python Intro - Basics

Before writing full programs, students need to understand the absolute fundamentals:

  • How to print text (“Hello World”)
  • Indentation rules
  • Colons after control statements
  • Basic syntax errors and how to spot them

Hello World

The simplest Python program:

print("Hello World")
Running the first Python program: Hello World

Notes:

Python Syntax Colours

Your editor highlights different parts of Python code to help you understand what each part does:

  • Blue — commands / keywords (e.g. print)
  • Terracotta — strings (text inside speech marks)
  • Green / Purple — numbers, variables, or other data depending on theme

Example: Your First Python Line


print("hello world")

This is the classic first program. It prints text onto the screen.

Breaking It Down

  • print → this is the command (lower‑case only).
  • () → parentheses (brackets) tell Python what you want to print.
  • "hello world" → this is a string (text). Speech marks show Python it’s text.
  • Put it together and you get: print("hello world")

Extra Notes

  • Python commands are lower‑case. Print will not work — print will.
  • Your theme colours different parts of the code. This helps you spot mistakes quickly.
  • In this example: blue = commands and text inside speech marks (strings) is terracotta.
  • Parentheses is the fancy word for brackets. You need them around the string.
  • Put it all together and you get: print("hello world")

Variables

Storing a user’s input and then using it in your program.

name = input("What is your name? ")
print("Hello", name)

This program asks the user for their name, stores it in a variable called name, and then prints a greeting using that variable.

Using Variables Inside Text

There are two common ways to include a variable inside a sentence:

  • Concatenation (joining text together): 
    print("Hello " + name)
  • f‑strings (recommended — cleaner and easier): 
    print(f"Hello {name}")

f‑strings let you insert variables directly into text using curly braces. They are the modern, preferred way to format strings in Python.

Common Misconception

Many beginners write code like this: 

name = "Mr Anderson"
print("Hello", name)

When they run it, it prints their name — so they think the program works. But the name is hard‑coded. It will always print the same name, even if someone else uses the program.

The correct version asks the user for their name: 

name = input("What is your name? ")
print("Hello", name)

This way, the variable name stores whatever the user types, not a fixed value.
