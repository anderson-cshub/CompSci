Input

Inputs are used when the user needs to enter their own data. To store this data, we use a variable.

A variable is a named piece of storage in memory. The value inside it can change — which is why it’s called a “variable”.

Exam tip: A variable can change. A constant cannot.

Inputs and Outputs — Writing Clean, Readable Code

Inputs and outputs are how your program communicates with the user. Good programs don’t just work — they are readable, well‑structured, and easy to maintain. This section covers best practice for naming variables, formatting output, and writing clean multi‑line code.

Meaningful Identifiers

A variable name should describe what it stores. Python allows several naming styles:

  • lowercase — simple variables: name, score
  • snake_case — Python’s preferred style: total_score, user_name
  • camelCase — common in JavaScript, allowed but not standard in Python: userName
  • kebab-case — not allowed in Python (hyphens are subtraction): user-name
  • UPPER_CASE — used for constants: MAX_SIZE = 10
# Good naming
user_name = input("Enter your name: ")
total_score = 0

# Constant (Python convention)
MAX_ATTEMPTS = 3

Using Input

input() always returns a string. Convert it when needed: 

age = int(input("Enter your age: "))
height = float(input("Enter your height in metres: "))

Output Formatting

Python lets you join text and variables in several ways:

  • Comma separator — easiest, adds spaces automatically.
  • Plus operator — joins strings only (must convert numbers).
  • f‑strings — modern, clean, recommended.
name = "Alice"
score = 42

print("Hello", name)              # comma separator
print("Score: " + str(score))     # plus operator
print(f"{name} scored {score}")   # f-string (best)

New Lines and Multi‑Line Output

Use \n for new lines, or triple quotes for multi‑line text. 

print("Line 1\nLine 2\nLine 3")

print("""
Name: Alice
Score: 42
Grade: A
""")

Avoiding Long Lines

If a line gets too long, break it across multiple lines using parentheses: 

message = (
    "This is a very long message that would normally "
    "go off the screen, but splitting it keeps the code tidy."
)

print(message)

Example — Clean Input/Output Program

user_name = input("Enter your name: ")
score = int(input("Enter your score: "))

print(f"\nResults for {user_name}")
print("----------------------")
print(f"Score: {score}")

This example shows clean naming, input conversion, f‑strings, and formatted output.
