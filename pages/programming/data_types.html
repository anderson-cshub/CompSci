Data Types

Integers (int)

Whole numbers with no decimal places. Used for counting, indexing, loops, and calculations. 

age = 15
score = 42

Floats (float)

Numbers with decimal places. Used for precise values such as measurements or averages. 

temperature = 19.8
average = 72.5

Strings (str)

Text data. Strings can contain letters, numbers, symbols, or spaces. 

name = "Mr Anderson"
welcome = "Hello, world!"

Booleans (bool)

True or False values. Used for decisions, conditions, and logic. 

is_logged_in = True
is_admin = False

Lists (list)

Ordered collections of values. Lists can store multiple data types. 

scores = [10, 20, 30]
names = ["Alice", "Bob", "Charlie"]

Type Casting

Converting one data type into another. Essential for input and calculations. 

age = int("15")
height = float("1.82")
text = str(99)

Understanding Python Data Types

Python is a dynamically‑typed language, which means you don’t need to declare data types explicitly — Python works them out automatically. However, understanding data types is essential for debugging, writing correct code, and succeeding in GCSE Computer Science exams.

Why Data Types Matter

  • They determine what operations you can perform.
  • They help prevent errors (e.g., adding a string to an integer).
  • They are tested directly in Paper 2 (programming and algorithms).
  • They appear in trace tables, dry‑runs, and exam pseudocode.

Common GCSE Mistakes

  • Forgetting to convert input using int() or float().
  • Using quotes around numbers, accidentally turning them into strings.
  • Confusing lists with strings (both are sequences).
  • Assuming booleans are written as true instead of True.
